CS-415 Information Security Assignment
Security Analysis of Expert Raah System

Section 1 — FYP Overview
Project Title: Expert Raah
System Purpose: Expert Raah is a web-based platform connecting clients with consultants in Education, Business, and Legal fields. It facilitates expert matching, profile management, and secure communication.
Target Users: Consultants (experts), Clients seeking advice, Platform administrators
Platform/Technologies Used: React (frontend), Node.js/Express (backend), Socket.IO (real-time), Passport.js, Helmet, CORS, JWT, Docker
Type of Data Handled: User profiles, authentication credentials, consultation records, payment information, real-time chat data

Section 2 — Identify Information Assets
Asset           | Description                        | Importance
-------------------------------------------------------------
User Accounts   | Consultant/client profiles         | High
Database        | Stores all platform data           | High
Admin Panel     | Manages users, content, settings   | High
API Services    | Backend endpoints for operations   | High
Socket Server   | Real-time communication            | Medium
Payment Gateway | Handles transactions               | High

Section 3 — Threats, Attacks & Vulnerabilities
Asset         | Possible Threat      | Possible Attack      | Vulnerability
--------------------------------------------------------------------------
User Accounts | Phishing, Insider    | Credential theft     | Weak passwords, session hijack
Database      | Malware, SQL Injection| Data breach         | Poor input validation
API Services  | DDoS, MITM           | Service disruption   | Unsecured endpoints
Admin Panel   | Device theft         | Unauthorized access  | Hardcoded credentials
Payment Info  | Fraud, Data theft    | Transaction tampering| No encryption
Socket Server | Replay attack        | Message interception | No authentication

Section 4 — CIA Triad Analysis
CIA Principle   | Risk in Your FYP           | Impact if Compromised
---------------------------------------------------------------------
Confidentiality | Data leakage, unauthorized access | User privacy loss, legal issues
Integrity       | Data tampering, fake records      | Incorrect decisions, fraud
Availability    | Server crash, DDoS, power failure | Service downtime, user frustration

Section 5 — Software Security Analysis
Vulnerability         | Location in Your System | Risk Level
-------------------------------------------------------------
Weak authentication   | Passport config, login | High
Hardcoded credentials | Admin panel, configs   | High
No encryption         | Payment, sensitive data| High
Poor input validation | API endpoints          | Medium
No logging/monitoring | Backend services       | Medium
Outdated libraries    | Backend/frontend       | Medium
No backup system      | Database               | High

Section 6 — Security Safeguards & Countermeasures
Security Goal   | Countermeasure           | How it protects system
---------------------------------------------------------------------
Confidentiality | Encrypt data at rest & transit | Prevents unauthorized access
Confidentiality | Role-based access control | Limits data exposure
Integrity       | Input validation         | Prevents data tampering
Integrity       | Audit logs               | Detects unauthorized changes
Availability    | Regular backups          | Enables recovery after failure
Availability    | DDoS protection          | Maintains service uptime

Section 7 — Reflection
1. Most critical CIA principle: Confidentiality, because the platform handles sensitive user and payment data.
2. Most realistic threats: SQL Injection, weak authentication, DDoS, session hijacking.
3. Security improvements before FYP-Final: Implement encryption, strengthen authentication, add logging/monitoring, update libraries, regular backups.
