CS-415 Information Security Assignment
Security Analysis of FYP-2 System

Section 1 — FYP Overview
Project Title: Expert Raah (with CNIC Validation & Verification Module)
System Purpose: Expert Raah is a comprehensive platform for expert consultation, guidance, and verification. The CNIC validation module automates identity verification using AI/ML and web technologies.
Target Users: Individuals seeking expert advice, consultants, administrators, and organizations requiring identity verification.
Platform/Technologies Used: Web (React, Node.js, Express), Python (AI/ML), Docker, Django, ONNX, YOLOv8. Main codebase in apps/backend and apps/frontend.
Type of Data Handled: User profiles, consultation records, CNIC images, personal information, authentication data

Section 2 — Identify Information Assets
Asset           | Description                  | Importance
-------------------------------------------------------------
User Accounts   | Registered users’ info, expert profiles | High
Database        | Stores user, expert, CNIC, and consultation data | High
Admin Panel     | Controls system settings, expert management | High
API Services    | Backend communication for consultation and verification | Medium
Cloud Storage   | Stores uploaded images/files (CNIC, documents) | Medium

Section 3 — Threats, Attacks & Vulnerabilities
Asset         | Possible Threat      | Possible Attack      | Vulnerability
--------------------------------------------------------------------------
User Accounts | Phishing, Insider    | Credential theft     | Weak passwords
Database      | Malware, SQL Injection| Data breach         | Poor input validation
API Services  | DDoS, Man-in-the-middle | Service disruption | Unsecured endpoints
Admin Panel   | Device theft         | Unauthorized access  | Hardcoded credentials
Consultation Data | Human error, privacy breach | Data leakage | Insufficient access control

Section 4 — CIA Triad Analysis
CIA Principle   | Risk in Your FYP           | Impact if Compromised
---------------------------------------------------------------------
Confidentiality | Data leakage, unauthorized access | User privacy loss, legal issues
Integrity       | Data tampering, fake records      | Incorrect decisions, fraud
Availability    | Server crash, DDoS, power failure | Service downtime, user frustration

Section 5 — Software Security Analysis
Vulnerability         | Location in Your System | Risk Level
-------------------------------------------------------------
Weak authentication   | Login module (apps/frontend/backend) | High
Hardcoded credentials | Admin panel (apps/backend)            | High
No encryption         | Data storage/transmission (apps/backend, CNIC module) | High
Poor input validation | API endpoints (apps/backend)          | Medium
No logging/monitoring | Backend services (apps/backend)       | Medium
Outdated libraries    | Backend/frontend                      | Medium
No backup system      | Database                              | High

Section 6 — Security Safeguards & Countermeasures
Security Goal   | Countermeasure           | How it protects system
---------------------------------------------------------------------
Confidentiality | Encrypt data at rest & transit | Prevents unauthorized access
Confidentiality | Role-based access control | Limits data exposure
Integrity       | Input validation         | Prevents data tampering
Integrity       | Audit logs               | Detects unauthorized changes
Availability    | Regular backups          | Enables recovery after failure
Availability    | DDoS protection          | Maintains service uptime

Section 7 — Reflection
1. Most critical CIA principle: Confidentiality, because Expert Raah handles sensitive user and expert data, including CNIC information.
2. Most realistic threats: SQL Injection, weak authentication, DDoS, privacy breaches in consultation data.
3. Security improvements before FYP-Final: Implement encryption, strengthen authentication, add logging/monitoring, update libraries, improve access control for consultation data.
