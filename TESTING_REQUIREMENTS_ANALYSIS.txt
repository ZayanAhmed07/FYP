1. OVERVIEW
================================================================================

Testing is a critical phase that must be performed BEFORE deployment to ensure
the system works as intended and meets all stated requirements. The testing 
process helps identify hidden errors and validates system quality.

Required Testing Types:
- Unit Testing
- Functional Testing  
- Integration Testing

All three types of testing are REQUIRED to ensure comprehensive system quality.

4. BUSINESS RULES TESTING
================================================================================

Methodology:
Decision table-based testing technique is used to test business rules.

The business rules were defined in FRs and Use Cases. Decision based testing 
uses a systematic approach where inputs and outputs are provided in tabular 
form. It is a precise and compact way to model complicated logic.

The form: Conditions and actions are used for test cases where conditions are
inputs and actions are outputs.

Detailed example is given in Appendix E (Decision Tables for SuperStores 
Marketing Policy).

Decision Table Methodology:
1. Identify Conditions & Values
   - Find the data attribute each condition tests and all attribute values

2. Identify Possible Actions
   - Determine each independent action to be taken for each policy

3. Compute Max Number of Rules
   - Multiply the number of values for each condition data attribute by each other

4. Enter All Possible Rules
   - Fill in the values of the condition data attributes in each numbered rule column

5. Define Actions for each Rule
   - For each rule, mark the appropriate actions with an X in the decision table

6. Verify the Policy
   - Review completed decision table with end-users

7. Simplify the Table
   - Eliminate and/or consolidate rules to reduce the number of columns


5. INTEGRATION TESTING
================================================================================

Definition:
Integration tests assess whether a set of classes that must work together do so
without error. They ensure the interfaces between different parts of the system
work properly.

At this point, classes have passed their individual unit tests, so focus is on
the flow of control as classes interact and messages are passed among them.

Integration testing uses the same general procedures as unit testing: develop
a test plan with series of tests, which in turn have a test case. Often done
by a set of programmers and/or systems analysts.

SUPERSTORES MARKETING POLICY #2:
• Repeat customers get free shipping
• New customers pay regular shipping on their first order
• Any customer who uses the SuperStore (SS) charge card gets a 5% discount

Decision Table:
┌──────────────────────┬────────┬────────┬────────┬────────┐
│ Condition            │ Rule 1 │ Rule 2 │ Rule 3 │ Rule 4 │
├──────────────────────┼────────┼────────┼────────┼────────┤

│ Used SS card?        │   Y    │   N    │   Y    │   N    │

│ 5% discount          │        │        │        │        │
└──────────────────────┴────────┴────────┴────────┴────────┘

Analysis:
- Rules 1 & 2 can be combined (Repeat customer = Y, SS card doesn't matter)
- Rules 3 & 4 remain separate (New customer, SS card matters for discount)


SUPERSTORES MARKETING POLICY #3:
• Repeat customers get free shipping
• New customers pay regular shipping on their first order. However, if a new

7. KEY TESTING PRINCIPLES
================================================================================

1. Testing Must Be Completed Before Deployment
   - All tests must pass before system goes live
   - Helps find hidden errors from user perspective

2. Comprehensive Testing Required
   - Unit testing validates individual components
   - Functional testing validates module functionality
   - Integration testing validates interface communication

3. Systematic Approach
   - Use test plans with documented test cases
   - Record expected vs actual results
   - Document pass/fail status

4. Business Rules Validation
   - Use decision tables for complex business logic
   - Test all possible rule combinations
   - Simplify tables by combining equivalent rules

5. Interface Testing Critical
   - Ensure proper data flow between modules
   - Validate message passing between classes
   - Test control flow in integrated systems
